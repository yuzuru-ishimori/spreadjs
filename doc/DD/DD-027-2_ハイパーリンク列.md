## 目的

列単位でセルクリックを「詳細画面への遷移材料」として利用側へ通知するハイパーリンク列を提供する（ReadyCrew 案件DB=クリックで案件詳細へ・stage2-backlog §3.6）。SDK は navigate せず `link-open` イベントで委ねる（親③）。

## 背景・課題

- 親 DD-027 の決定事項（転記・2026-07-21 ユーザー確定）:
  - **③**: 値は **string 1本**（表示テキスト＋URL の分離モデルなし）。クリック時は **navigate せず公開イベント `link-open`（rowId/columnId/value）が既定動作**。opt-in の既定 open を提供する場合は **http/https のみ**（javascript:/data: 等は診断 warn＋無視）。クリックでも **activeCell 移動は従来どおり**＋`link-open` 発火（選択を奪わない）。編集手段（F2/ダブルクリック/直接入力）は維持。
  - **⑥**: 列タイプは両モード共通の mount オプション `columnTypes`（mount 時固定）。cell-commit/setData の値契約不変。新規公開イベントは `link-open` のみ。設定一致は利用側責務。
- DD-027-1 で成立済みの基盤（2026-07-21 実調査・二重設計しない）: `packages/grid/src/column-types.ts`（公開型 `GridColumnType`=現状 `GridSelectColumnType` のみの union・Internal `createColumnTypeRegistry`・`ColumnTypeConfigError` fail-fast）／`GridCommonMountOptions.columnTypes`＋公開 error 語彙 `column-types-invalid`（mount 時 fail-fast 配線済み）／mount-controller の registry 生成・editor 経路 validator（link 列は制約なし=素通り）／playground `?select=` 配線・E2E `column-types-select.spec.ts` の先例。
- クリック経路の実調査（`mount-controller.ts`）: pointerdown=リサイズ先取り→hitTest→（Shift+クリック分岐）→`selectionCtrl.clear()`→`editor.pointerdownCell(cell)`→Navigation なら**即ドラッグ選択開始**（pointer capture）。pointerup=`finishSelectionDrag(confirm=true)`→`endDrag`（同一セルなら単一選択のまま）。dblclick=`editor.doubleClickCell`（既存値編集）。非ドラッグ pointermove はセル領域で**意図的に hitTest を省略**（cheap path・予算保護）。
- 描画の実調査（`packages/render/src/base-layer.ts`）: セル文字は queryRange visitor で描画し、**列 index フック `isWrapColumn(col)` の先例**あり。文字色は数値/通常の2択・下線描画は無い。左寄せ文字列は右隣空セルへオーバーフローする（リンク列でこれを許すと「描画とクリック領域の不一致」が生じる）。

## 📐 実装前詳細化（親 Phase 0 判定=要。2026-07-21 実施済み・👀 ユーザーレビューは実装開始前の確認ゲートで）

**結論: 既存の pointerdown 経路（activeCell 移動・ドラッグ選択開始・IME/編集裁定）は無変更のまま、「クリック完了」判定を pointerdown〜pointerup の候補追跡として**上乗せ**する。editor-state-machine・ime-editing-session・commit-bridge・常駐 textarea・editor validator は無改変＝T1 非該当。**

- **クリック裁きの設計（候補追跡方式）**:
  1. pointerdown（button=0・`event.detail===1`＝連打の2打目は候補化しない）で hit セルが**リンク列・値非空・Navigation 位相・非 composing** なら link 候補 `{rowId, columnId, value, cell}` を記録する（既存処理はそのまま実行＝activeCell 移動・ドラッグ選択開始）。
  2. ドラッグで focus セルが開始セルから離れたら（`selectionCtrl.updateDrag` が別セルへ動いたら）候補を破棄する（=ドラッグ選択は発火しない）。
  3. pointerup（`finishSelectionDrag(confirm=true)` の直後）で候補が生きていれば `link-open` を emit する（rowId/columnId は文字列・R7。**SDK は navigate しない**）。pointercancel/capture 喪失は破棄。
  - この方式なら既存分岐（Shift+クリック・リサイズ先取り・編集中クリック=確定して移動/pendingNavigation）は候補化の条件を満たさず、**従来挙動が定義から保存**される（Editing/Composing 中のクリックで発火しない=AC8）。
- **dblclick との共存（新規論点→確認ゲートへ）**: ダブルクリック編集は従来どおり成立するが、**1打目のクリックで link-open が1回発火する**（`detail===1` により2打目では発火しない）。抑止にはクリック確定の遅延（dblclick 待ち）が必要で操作感を損なうため採らない。SDK は navigate しないため実害は利用側ハンドラの実装次第（リンク列の編集は F2/直接入力を推奨と文書化）。
- **defaultOpen（opt-in・親③の「提供する場合」を提供する）**: `GridLinkColumnType.defaultOpen?: boolean`（既定 false）。true のとき link-open 発火に加えて SDK が open する。**絶対 URL かつ http/https のみ**（`new URL(value)` パース＋protocol 検査。相対・非URL・javascript:/data: 等は **open せず診断 warn**）。open は `window.open(url, '_blank', 'noopener,noreferrer')`。link-open イベント自体は defaultOpen の成否に関わらず常に発火（利用側が観測可能）。
- **モジュール構成（新規0・変更4＋配線）**:
  - `packages/grid/src/column-types.ts` — `GridLinkColumnType{type:'link', defaultOpen?}` 追加・`GridColumnType` union 拡張・registry へ `isLinkColumn`/`getLinkType` 追加（`unsupported-type` 分岐の解除・fail-fast 検査は既存機構を流用）。
  - `packages/grid/src/index.ts` — `GridEvent` へ `{type:'link-open', rowId, columnId, value}` 追加・`GridLinkColumnType` re-export・columnTypes doc 更新。
  - `packages/grid/src/mount-controller.ts` — 候補追跡（上記1〜3）・defaultOpen 実行・**wrapColumns×link 併用の mount 時 fail-fast**（描画契約が両立しないため=検討内容）・リンク列 hover の `cursor:pointer`（**列単位の軽量判定のみ**=非ドラッグ pointermove の cheap path を壊さない。予算退行が出たら対象外へ縮退可）。
  - `packages/render/src/base-layer.ts` — `isLinkColumn?: (colIndex)=>boolean` フック（isWrapColumn と同型）＋リンク色（`linkText` を `BaseLayerColors` へ追加・既定 #1a73e8 系）＋下線（fillRect 1px）＋**リンク列はオーバーフロー対象外**（自セル内 fitText クリップ=クリック領域と描画を一致させる。左外流入の origin からも除外）。
  - 配線 — mount-controller から base-layer/document-view へ isLinkColumn を渡す（isWrapColumn の配線と同型）。`apps/playground/src/integration/main.ts` へ `?link=col-4` パラメータ（`?select=` と同方式）。
- **エッジケース**: 数値に解釈される値（`123`）もリンク列では**リンク描画（左寄せ・下線）を優先**（数値右寄せより列タイプが勝つ・表示文字列は不変）／空セル=装飾なし・発火なし／tombstone 行は hitTest に現れない（既存）／pointerup 時に対象行が構造変更で消えていても link-open は「クリック時に捕捉した値」で発火してよい（navigate しない通知のみ・文書は触らない）。
- **テスト観点**: 候補追跡の裁定表（クリック/ドラッグ/Shift+クリック/dblclick/編集中/composition 中 × 発火有無）＝unit（純関数化した候補判定）＋E2E／URL 検査（http/https/javascript:/data:/相対/非URL）＝unit／描画（色・下線・非オーバーフロー）＝E2E スクリーンショット or ピクセル読み。

## 検討内容

- **発火タイミング**: pointerdown 発火は「ドラッグ選択のつもりが遷移」誤爆を生むため不採用。**pointerup・同一セル・detail=1** を採用（📐）。ブラウザ `click` イベントは「同一要素」判定でありセル単位判定にならないため自前追跡とする。
- **wrapColumns×link 併用**: wrap（折り返し・自動行高）とリンク（単行 fitText・下線）の描画契約が両立しないため mount 時 config error（`column-types-invalid`）で不可とする（最小・誠実。要求が出たら描画契約を定義して解除）。
- **キーボード/タッチ**: **対象外を明記** — Enter は既存の確定/下移動を維持（キーボードでリンクを開く手段は提供しない。要求が出たら Ctrl+Enter 案を拡張点として親 P-07 メモへ）。タッチは Tier 1（Win Chrome/Edge デスクトップ）の製品境界外（roadmap §6）。
- **DA（親 Phase 0 より）**: クリック=遷移の列で「セルを選ぶだけ」の操作が link-open を伴う（利用側が即遷移を実装すると選択目的のクリックができない）→ 通知のみ・ドラッグ/Shift+クリック/キーボードで発火しない設計がその逃げ道（利用側ガイドへ記載）。
