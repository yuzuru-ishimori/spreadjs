## 目的

`columnCaptions`（列見出しの業務名置換）と `columnDisplayFormats`（数値・日付の表示書式）の mount オプション2点で、「データと列スキーマを渡すだけで業務の表として読める」明細閲覧を成立させる（親DD-033 の第2子・AC4〜5 担当）。書式は **Canvas 描画テキストのみ**を整形し、文書・コピー・commit の契約面は一切変えない。

## 背景・課題（親確定値の転記・2026-07-21 ユーザー最終確定）

- **要確認③（キャプション）**: `columnCaptions`（ColumnId 文字列→表示名）。指定列はヘッダーの A/B/C を置換描画・未指定列は従来どおり。行番号ヘッダーは現状維持・2段ヘッダーは対象外。
- **要確認④（書式語彙）**: 自前ミニ書式 `columnDisplayFormats`（ColumnId→書式）。number=`{ grouping?, decimals?, percent?, prefix?, suffix? }`・date=固定トークン（`YYYY`/`MM`/`DD`/`HH`/`mm`/`ss`）のパターン文字列。Excel 書式文字列互換・Intl ロケール書式は v1 対象外（拡張点メモのみ残す）。
- **要確認⑤（適用面）**: 書式は **Canvas 描画テキストのみ**整形。cell-commit/setData round-trip・コピー TSV・columnFormats（DD-027-3）の完全一致・編集ドラフト文字列は従来の表示文字列（raw）のまま＝**契約不変**。
- **要確認⑥（mount 契約）**: 両モード共通・mount 時固定（wrapColumns/columnTypes と同運用）。不正設定は mount 時 fail-fast（phase=config・新 error code は本子DDで確定〔`column-types-invalid` と同経路〕→ 決定事項）。共同編集モードでの全クライアント設定一致は利用側責務。
- 表示整形は計画書で R-18 として保留されていた領域（`document-view.ts` cellScalarToDisplay のコメント）。本子DDが v1 を提供する。

## 📐 実装前詳細化（起票時 2026-07-21 実施済み・👀 レビューは親セッションの仕様確認ゲートで）

- **書式純関数の置き場**: `packages/grid/src/display-format.ts`（新規・`format-rules.ts`〔DD-027-3〕と同流儀＝DOM/内部 package 非依存の TDD 対象）。公開型（`GridColumnDisplayFormat`）を定義するため grid 側に置く（R7: render の型を import しない・render へは**構造フック注入**のみ）。`packages/render` は依存ゼロのまま。
- **base-layer 接続点**（`packages/render/src/base-layer.ts`）:
  - ヘッダー: `drawColHeader`（L462〜464）が `columnLabel(col)` を直描画 → 新フック `columnHeaderLabel?: (colIndex) => string`（既定=columnLabel）で置換。**キャプションは `columnHeaderRect` 幅で fitText**（現行 A/B/C は短く顕在化しないが、長い業務名は隣接ヘッダーへ重なるため省略記号でクリップ）。
  - セル: `drawPaneValues` の queryRange visitor（L366〜）冒頭に新フック `formatCellText?: (colIndex, rawValue) => string`（getCellStyle と同型・未指定なら恒等）で display 文字列を確定し、以降の描画（バッジ/リンク/数値/wrap/オーバーフロー）は display を使う。**判定は raw のまま**: `getCellStyle(col, raw)`（columnFormats 完全一致=raw・要確認⑤）・`isNumericCell(raw)`（書式済み "1,234.5" は NUMERIC_RE 不一致のため、右寄せ判定を raw で行わないと数値の右寄せが壊れる）。Pass 2 左外流入（L400〜427）も origin 列の書式で display を適用。
- **text-cache 整合**: `TextMetricsCache` は「font×文字列（×幅）」の**内容キー**（text-cache.ts L52/L68/L100）で、セル位置や raw 値に紐づくキャッシュは存在しない → 書式済み文字列は**別キーとして自然に共存**。キー設計変更・無効化とも**不要**（書式は mount 時固定＝raw→display 対応が不変）。エントリ増は既存上限 8192 の FIFO/LRU 内。
- **mount 時プリコンパイル**（DD-027-3 前例）: `buildColumnTypeRegistry`（mount-controller.ts L1030〜）内で `compileDisplayFormats(options.columnDisplayFormats, options.columnCaptions, columnOrder)` を呼び、検証＋コンパイル済み書式関数（列→closure）を生成。`hasAny()` cheap-path で、両オプション未指定なら base-layer へフック非束縛＝描画コスト増ゼロ。
- **number 入力値判定**: raw が base-layer と同一の数値形（`NUMERIC_RE`=`^-?\d+(\.\d+)?$`）のときのみ適用・非数値 raw は **raw 素通し**。整形は**文字列ベースの十進演算**（float/toFixed 非経由＝2進誤差なし・決定的）: percent=小数点2桁右シフト→decimals=half-up 丸め→grouping=3桁カンマ→出力 `prefix + 本体 + ('%') + suffix`（要確認3）。
- **date 入力値判定**: raw が `YYYY-MM-DD` または `YYYY-MM-DD[T|空白]HH:mm(:ss)` に正規表現一致した場合のみ適用（CellScalar date kind の正準文字列 YYYY-MM-DD を包含）。**フィールド直取り**（Date オブジェクト/タイムゾーン変換を経由しない＝環境非依存・決定的）。非受理形（数値シリアル・和暦等）は **raw 素通し**・セル単位の診断は出さない（スパム防止・要確認4）。時刻フィールド欠落 raw に HH/mm/ss トークンは `00` を埋めず**適用不可=raw 素通し**とする（嘘の 00:00 を描かない）。
- **fail-fast の新 error code**: **`column-display-invalid`**（`GRID_ERROR_CODES` へ追加・phase=config）。`DisplayConfigError` を `buildColumnTypeRegistry` の catch で写像（`column-types-invalid` と同経路）。既存 code へ相乗りしない理由: キャプション/表示書式は列「タイプ」ではなく、意味を広げると障害切り分けが濁る。検証対象=未知列・空/空白キャプション・不正 type・decimals 非整数/0〜20 外・date pattern 空/既知トークン皆無。
- **併用相互作用**: wrapColumns 同一列=**fail-fast**（自動行高が raw の wrapLines で計算され描画だけ書式済みになる構造不整合・要確認1）／link 列=**fail-fast**（表示整形とクリック対象 raw URL の乖離・要確認2）／select 列=許可（構造不整合なし・候補/検証は raw のまま=JSDoc 注意・要確認2）／columnFormats=併用可（match は raw 完全一致のまま・バッジのチップ文字とチップ幅は display で描く/測る）／readOnly=直交（併用可・親AC7 デモ構成）／auto-fit=内容幅測定（mount-controller L687）を display で・headerLabelWidth（L695）をキャプションで測る（描画と測定の共有=DD-027-3 Fable P3 の教訓踏襲）。
- **50k 行スクロールコスト**: 書式適用は可視セルのみ（visitor 内・約 40 行×25 列 ≈ 1,000 セル/フレーム）×文字列演算 O(桁数)＝µs オーダー → redraw 予算 12ms に対し余裕見込み。パターン解析は mount 時1回・高コスト側の measureText は text-cache が書式済み文字列キーでヒット。raw→display の列別 memo は v1 では入れず、親 Phase 3 headed 計測（AC7）で予算超過時の対策カードとする。

## 検討内容

- 型は columnTypes 前例の判別 union: `GridColumnDisplayFormat = { type: 'number', grouping?, decimals?, percent?, prefix?, suffix? } | { type: 'date', pattern: string }`。date pattern はトークン以外の文字をリテラル素通し（`YYYY/MM/DD` の `/` 等）。
- 描画テキスト差し替えの設計は「フックで display を確定し、判定（数値右寄せ・書式 match・wrap/link 分岐）は raw」の1点に集約（📐）。document-view・chunk-store・clipboard・commit-bridge は無改変＝契約不変（⑤）が構造的に成立（コピー/commit は `view.cellDisplay`=raw を読む既存経路のまま）。
- 拡張点メモ（v1 対象外・実装しない）: Excel 書式文字列互換・Intl/ロケール書式（`Intl.NumberFormat`/`DateTimeFormat` 委譲）・負数の色/括弧表現・数値シリアル日付・2段ヘッダー。複数実案件で要求されたら別DDで I/F 拡張を判断。

## 要確認（各項目に既定案。親セッションがユーザーへ提示）

> **2026-07-22 仕様確認ゲートで1〜4すべて既定案どおり確定**（決定事項参照）。

1. **wrapColumns との同一列併用** — 既定案: **fail-fast**（`column-display-invalid`）。行高計算（raw の wrapLines）と描画（書式済み）が乖離する構造不整合のため（wrap-link-conflict=DD-027-2 前例）。対案: wrap 列では書式を黙って無効化（サイレント破壊=非推奨）／行高計算にも書式適用（document-view へ書式依存が漏れる）。
2. **列タイプとの併用** — 既定案: link 列=**fail-fast**（表示は整形・クリックは raw URL の乖離を防ぐ）・select 列=**許可**（構造不整合なし。候補・検証・ドロップダウン表示は raw のまま＝JSDoc に注意記載）。対案: link も許可（描画のみ整形）。
3. **number 整形の細目** — 既定案: 丸め=half-up（文字列十進・decimals 既定=raw の桁のまま〔丸めなし〕）・grouping 記号=カンマ固定（ロケール非依存）・percent は decimals 丸めの前に2桁シフト・出力順 `prefix + 本体 + ('%') + suffix`。
4. **date の受理形** — 既定案: `YYYY-MM-DD` と `YYYY-MM-DD[T|空白]HH:mm(:ss)` のみ（フィールド直取り・タイムゾーン非経由）。非受理 raw・時刻欠落 raw への時刻トークンは raw 素通し（セル単位診断なし）。対案: 受理形を広げる（`YYYY/MM/DD` 入力等・v1 では絞る）。
