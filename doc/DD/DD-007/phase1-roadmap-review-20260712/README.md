# ChatGPT相談パック: Phase 1 DDロードマップ草案の妥当性レビュー（2026-07-12）

> Phase 0（技術PoC）の Go/No-Go を判定中（DD-007）。その先の **Phase 1 ＝ 製品成熟 Stage 1「社内SDK Alpha」までの縦切りDD計画**を草案として組んだ。**この計画が妥当か**を ChatGPT にレビューしてもらうための一式。
> ChatGPT はこのリポジトリを直接読めないため、**アップロードするだけで文脈が完結する**ようにファイルを揃えてある。

## 使い方

1. ChatGPT（推論モデル推奨。o系／GPT-5系など「よく考える」設定）を開く。
2. 下記ファイルを**添付**する（`03` がレビュー対象の本体、`02` が観点、`04〜06` は裏付け資料）。
3. `01_プロンプト.md` の中身を**そのまま貼り付け**て送信する。
4. 返ってきた助言は、**計画本体 `doc/plan/phase1-dd-roadmap.md` と DD-007 へ還流**する。外部レビューは手動でDDへ戻す運用（プロジェクトの既定）。記録先: `doc/DD/DD-007/chatgpt-review-YYYYMMDD-*.md`。

## この相談で決めたいこと（要約）

- 草案の **到達目標（Stage 1 SDK Alpha）と縦切りDDの切り方・順序・依存**は妥当か。
- 計画書 §19 の **Phase 1（単一利用者）/Phase 2（共同編集）境界**と、縦切り目標（最初のDDで共同編集保存まで）の**整合のさせ方（DD-010単一利用者／DD-011共同編集保存の分割）は健全か**。
- **Stage 1 Alpha の最小到達ライン**（どのDDまでを「Alpha成立」とするか。特にデータ損失回避＝reconnect堅牢化 DD-015 を必須にするか）。
- 織り込んだ**密度レジーム（Risk Class A/B/C・例外ベース人間ゲート・証跡5点圧縮）**が Phase 1 で機能するか、抜け・危険はないか。
- **見落としている縦切り・リスク・順序の誤り**。
