# ChatGPT相談パック: 開発プロセスの「密度」をこれからどうするか（2026-07-12）

> Phase 0（技術PoC）が想定より重く進んでいる。**これからの進め方を、今の密度のまま行くか／もう少し薄くできるか**を ChatGPT に相談するための一式。
> ChatGPT はこのリポジトリを直接読めないため、**アップロードするだけで文脈が完結する**ようにファイルを揃えてある。

## 使い方

1. ChatGPT（推論モデル推奨。o系／GPT-5系など「よく考える」設定）を開く。
2. 下記4ファイルを**添付**する（`02` が本体、`03〜05` は裏付け資料）。
3. `01_プロンプト.md` の中身を**そのまま貼り付け**て送信する。
4. 返ってきた助言は、**DD-007（Phase 0 Go/No-Go・Phase 1正式バックログ確定）へ還流**する。外部レビューは手動でDDへ戻す運用（プロジェクトの既定）。記録先の例: `doc/DD/DD-007/chatgpt-review-20260712.md`。

## この相談で決めたいこと（要約）

- 今の密度（DDごとに DA＋Codex xhigh＋外部レビュー＋実機ゲート、細かいPhase分割、厚い証跡）は **Phase 0（技術検証）では妥当かもしれないが、Phase 1以降（製品構築）でも同じ密度で回すべきか**。
- 薄くするなら**どのレバーを・どのPhaseで・どこまで**。品質を落とさないための歯止めは何か。
- 逆に「今のままが良い」なら、その根拠と、遅さを受け入れる代わりに得ているもの。
