## 目的

Excel 同様に列ヘッダー・行ヘッダーの境界をドラッグして列幅・行高を変更できるようにする。設定は **view-local**（他ユーザーへ即時同期しない）とし、変更イベント＋mount 初期値の公開APIで**利用側アプリが保存・復元**できるようにする（保存先を共有にすれば F5 リロードで他ユーザーへも反映＝ユーザー確認済みの要件）。

## 決定事項（仕様確定済み）

- **D1 保存範囲 = view-local＋利用側保存API**（ユーザー確定 2026-07-15）。protocol・snapshot・Operation は無改変。全ユーザーリアルタイム共有は Stage 2 バックログへ（stage2-backlog.md に記録済み）。
- **D2 公開API（Experimental 0.x 追加・CHANGELOG 必記）**:
  - `GridMountOptions.columnWidths?: Readonly<Record<string, number>>`（ColumnId→px・初期 override）
  - `GridMountOptions.rowHeights?: Readonly<Record<string, number>>`（RowId→px・初期 override）
  - `GridEvent` に `{ type: 'layout'; columnWidths: Record<string, number>; rowHeights: Record<string, number> }` を追加。**ドラッグ確定時（pointerup）に発火**・内容は override のみ（既定値の列/行は含まない）。利用側はこれを保存し、次回 mount の初期値に渡す（F5 反映の実現手段）
- **D3 操作仕様（Excel 風）**: 列ヘッダー右端／行ヘッダー下端の **±4px** でカーソルを `col-resize`/`row-resize` に変更。左ボタンドラッグで変更・ドラッグ中ライブ再描画。最小 列 20px／行 16px・最大 2000px でクランプ。**ダブルクリック auto-fit は対象外**（Stage 2 バックログ記録済み）。
- **D4 実装方式**: 既存資産をそのまま使う — `Axis.setSizeById`（`packages/render/src/axis.ts`・実装済み）＋`ViewportTransform.hitTest`（`column-header`/`row-header` 判別済み）＋`columnHeaderRect`/`rowHeaderRect`。リサイズ後は `markViewportDirty` → 再描画＋（編集中なら）`refreshPlacement` で textarea 追従。
- **D5 IME 不変の維持**: composition 中のリサイズも textarea の value/selection/DOM 親には触れない（`place()` のみ＝I-3 の許容範囲・既存 refreshPlacement 経路）。リサイズ操作は編集状態機械に**イベントを流さない**（pointerdown をリサイズ層で先取りし editor へ渡さない）。
- **D6 features.json 更新**: 完了時に `apps/showcase/src/features.json` へ「列幅・行高の変更」を提供中で追加＋デモシナリオ追記（AGENTS.md 義務）。
