## 目的

ドラッグ／Shift+クリック／Shift+矢印による矩形範囲選択と、範囲クリア（Delete＝blank 敷き詰めの**原子的 SetCells**）を提供する。あわせて「選択レンジ読み取り」「範囲→原子 SetCells 生成（上限検査込み）」を内部 API として確立し、DD-020-2（paste/cut）・DD-020-3（Undo）の土台にする。

## 背景・課題（親 DD-020 §背景の該当分＋追加調査）

- **幾何は既存**: `packages/selection/src/selection.ts`（`rangeFromAnchorFocus`/`rangeContains`/`singleCell`）。
- **描画も既存**: `packages/render/src/overlay-layer.ts` は `frame.selection`/`frame.dragRange`（`CellRange | null`）の矩形描画（pane 交差・fill/stroke）を実装済み。**欠けているのは選択状態の所有者と入力配線**＝`packages/grid/src/mount-controller.ts` は `selection` へ `singleCell()` しか渡さず、pointerdown はドラッグなし・`packages/ime/src/navigation.ts` に Shift+矢印拡張なし。
- **activeCell の所有**は `packages/ime/src/editor-state-machine.ts` に一本化済み（DA #2）。選択レンジの所有者は不在。
- **単一セル Delete は実装済み**（Navigation Delete・S-A4・`packages/grid/src/ime-editing-session.ts`）。範囲版が必要。
- SetCells の原子性（I-5）・beforeRevision 規約（`captureEditStartRevision`・未書込=0）は既存資産（親 §背景）。

## 検討内容

- **所有者設計（2案・Phase 1 の 📐 で確定）**:
  - 案X（初期方針）: grid 層に `selection-controller` を新設。editor-state-machine へは `getActiveCell()`/`getPhase()` の読み取りのみ依存し、**Shift+矢印・ドラッグは状態機械の前段（mount-controller の keydown/pointerdown 裁定）で消費**して状態機械へ流さない（遷移追加なし・挙動保存・CG-1 資産無変更）。
  - 案Y: editor-state-machine 内へ selection 状態を追加（遷移追加=Codex xhigh 昇格・CG-1 資産の変更）。
  - 案X で activeCell との整合（anchor 固定）が崩れる場合は停止して再判断（親ヘッダ昇格条項）。
- **Shift+矢印セマンティクス**: anchor 固定・focus 端のみ拡張（Excel 準拠）。activeCell は anchor に留まる。
- **選択解除**: 通常矢印移動・クリック・Escape・編集開始（printable/F2/ダブルクリック）で単一選択へ戻る。
- **範囲クリアの changes 構成**: 上限判定は**範囲セル数**（体感と一致）で行い、SetCells の changes には**非空セルのみ**含める（結果同一・operation 最小化。beforeRevision はクリア実行時点の committed `lastChangedRevision`＝`captureEditStartRevision` 規約）。
- **対象外（既定案・オーケストレータ確定対象）**: (c) Ctrl+A 全選択＝対象外（100,000 セル上限拒否との UX 衝突回避・将来拡張）。ドラッグ中の viewport 外 autoscroll＝対象外（C級拡張・viewport 内ドラッグのみ）。

- 本子DD: 所有者は案X 初期方針・範囲クリアは非空セルのみ changes・Ctrl+A / autoscroll 対象外。
