## 目的

利用者成果=「**日本語で文字列/数値/日付をセルへ連続入力し、Enter/矢印で移動・確定できる**」を、PoC資産の実抽出の上で製品品質にする。IME状態機械（DD-002/005）を `@nanairo-sheet/ime` へ、selection/navigation を `@nanairo-sheet/selection` へ実抽出し、型変換（標準セット）とローカルOperation（サーバーなし SetCells・hash 決定性）を確定経路に実装し、**CG-1（実機IME）を解除**する。

## 背景・課題

- DD-009 資産台帳の割当: `ime/editor-state-machine`＝Adopt・`integration/ime-editing-session`＝Adopt・`ime/event-recorder`＝Harden（→`@nanairo-sheet/ime`）／`grid/navigation`・`pocb/selection`・`grid/geometry`＝Harden（→`@nanairo-sheet/selection`）／`integration-editor`・`editor-placement`・`document-view`・`commit-bridge`＝Adopt/Harden（grid 側素材）。Q5=`resident-textarea` は integration-editor 一本化後に不使用化確認。
- **CG-1 は DD-005 Phase 5 のユーザー判断（実機なしクローズ）の回収**: 「新 integration-editor アダプタ×実IME候補ウィンドウ・順序A/B の実機記録」が未取得（roadmap §8）。未解除は Alpha 不可・期限=Facade公開前。
- CellScalar は `blank/string/number` のみ（`packages/core/src/operations.ts`）。日付入力に `date` 追加＝データ形式変更（codec/hash/apply/validate へ波及）。DD-013（OCC）・DD-014（snapshot）より前に確定すべき変更のため本DDで実施し、ADR-012（LocalDate）をドラフト→Codex 承認で Accept。
- DD-011 設置の `tests/invariants/ime` は最小ケース（S-F2/F3）のみ → §2.3 IME不変条件6項目へ実充足する。
- boundary baseline のうち `apps/playground/src/{ime,grid,integration}` 由来の R1 entries（owner=DD-012 分）を本DDで縮退。

## 検討内容

- **Phase構成の方針（親DD指示）**: 実装ロジック部（Phase 1〜2）と Codex レビュー（Phase 3）までを自動実装可能にし、**人手必須の CG-1 実機ゲートを Phase 4 に分離**する。Phase 3 完了時点で「実機ゲート待ち」として停止できる。
- **型変換の位置**: 確定時の commit 経路（commit-bridge の parse）で行い、composition 中の状態機械・textarea には手を入れない（IME不変条件維持）。
- **標準セットの具体化**: 受理書式の具体一覧（数値: 半角/全角・桁区切り・負数・小数／日付: `YYYY-MM-DD` 等西暦一般書式）は Phase 2 の実装前詳細化で表として固定し、Codex xhigh で検証する（ユーザー確定④の運用）。偽陽性変換（例: 電話番号・型番の数値化）を DA 観点に含める。
- **順序A/B の実機記録**: 既定=DD-002 実機手順の踏襲（event-recorder＋`ui/trace-panel` を Q6 どおり dev tool 硬化して採取）。変更が必要なら要確認⑥として再提示。

## 決定事項

- 抽出は台帳の採用方針に従い**規約・設計の新規発明をしない**（挙動保存＋既存テスト green 維持）。
- CellScalar `date` は `{ kind: 'date'; value: 'YYYY-MM-DD' }`（LocalDate・計画書 D-08。JS `Date` を正規値にしない）。
- ローカルOperation=サーバー接続なしで SetCells がローカル文書へ適用され documentHash が決定的であること。共同編集同期・OCC・サーバー送信の変更は**一切含めない**（DD-013）。
- 新 workspace（ime/selection）追加は package-lock 変更を伴う → 着手前に `git status`＋並行セッション確認。
