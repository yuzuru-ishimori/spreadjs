# ChatGPTへ貼り付けるプロンプト

---

あなたには、**ソフトウェア開発のプロダクト計画・デリバリー戦略・アーキテクチャのアドバイザー**として相談に乗ってほしい。「開発者ほぼ1名＋AIエージェント（設計書起票・実装・コードレビューをAIが担当）」という体制で、リアルタイム共同編集スプレッドシート基盤（TypeScript・Canvas描画・日本語IME・サーバー主導共同編集）を開発している前提で答えてほしい。

## 依頼

**Phase 1 の開発ロードマップ草案（添付 `03_計画本体`）が妥当かをレビューしてほしい。** これは、技術PoC（Phase 0）の Go/No-Go 判定の直後に着手する Phase 1 を、**製品成熟 Stage 1「社内SDK Alpha」（別の社内プロジェクトから npm で利用可能にする段階）** までの**縦切りDD（利用者が一連の操作を完了できる単位）**として組んだものだ。

まず `03_計画本体` を通読し、`02_レビュー観点と論点` の論点に沿って、必要に応じ `04`（Phase 0 成果と憲章 Stage 1）・`05`（開発計画書のフェーズ構成）・`06`（直前の密度レビュー結論＝この草案が織り込んだプロセス方針）を参照して回答してほしい。

**なお Phase 0 の判定は既に確定した＝条件付きGo**（`04` §2.5）。実機IME検証（CG-1）を筆頭に **CG-1〜6** を解除条件に付す。したがって「Go前提のたたき台」ではなく、**確定した条件付きGo の条件を、この草案が各DDへ正しく割り付けているか（論点E）**も併せて評価してほしい。

## 出してほしい成果物（この順で）

1. **総合判定**: この Phase 1 草案は「そのまま進めてよい／要修正／組み直すべき」のどれに近いか。最初に結論を述べてから根拠を展開してほしい。
2. **到達目標（Stage 1 SDK Alpha）の妥当性**: 到達目標の置き方・移行条件 S1-1〜S1-5 への割付は適切か。Stage 1 に「共同編集の最小部分」を含める判断（単一利用者コアだけで止めない）は妥当か。
3. **フェーズ境界の整合（最重要論点）**: 計画書 §19 では Phase 1＝単一利用者・Phase 2＝共同編集/永続化だが、縦切りの目標（最初のDDで「日本語入力→共同編集保存」まで）は両者にまたがる。草案はこれを **DD-010（単一利用者コア・保存なし）と DD-011（共同編集保存）に分割**して整合させている。この整合のさせ方は健全か。**本来 Phase 2 のリスクを Phase 1 の軽量プロセスで扱ってしまう危険**は残っていないか。
4. **縦切りDDの切り方・順序・依存**: DD-009（基盤）〜DD-018（Stage 1移行判定）の分割粒度・順序・依存関係は妥当か。**過積載（1つのDDに詰め込みすぎ＝前回 DD-005 が重くなった失敗）**の兆候はないか。逆に**細かすぎて縦切りになっていない**箇所はないか。
5. **Stage 1 Alpha の最小到達ライン**: どのDDまでを「Alpha成立」とすべきか。特に**データ損失回避（切断・再接続で入力を失わない＝DD-015 reconnect堅牢化）を Alpha 必須にすべきか、強く推奨に留めてよいか**。行操作（DD-016）・数式（DD-017）を Stage 1 に含めるか Stage 2 へ送るかの判断材料も。
6. **密度レジームの妥当性**: 草案が織り込んだ Risk Class A/B/C・例外ベースの人間ゲート・証跡5点圧縮・変更トリガー方式の実機ゲート（`06`）は、Phase 1 で実際に機能するか。**軽量化しすぎて危険な箇所／逆に重いままの箇所**はどこか。
7. **見落とし**: この体制・このプロダクトで、草案が**見落としている縦切り・リスク・順序の誤り・前提の飛躍**。逆に**過剰に心配している**点。

## 制約・お願い

- **抽象論（「アジャイルに」等）ではなく、添付の実際の計画に即して**具体的に。トレードオフは必ず明示（「速くなるが○○を失う」）。
- **体制の前提**: 実装・レビューの多くをAIが担い、希少資源は**人間（ほぼ1名）の確認時間とAPI/サブスク枠**。「人を増やす」は選べない。
- **成熟段階の前提**: 到達目標は「本番完成」ではなく「別の社内アプリから使える SDK Alpha（API は 0.x で変更可）」。過剰な API 早期固定は避けたい。
- 断定できないところは仮定を明示した上で述べてよい。
- **最終的に、この草案へ具体的に反映できる修正提案（DDの分割変更・順序変更・Alpha必須ラインの確定・密度調整）を箇条書きで**まとめてほしい。
