## 目的

列単位で「決められた値だけ選択入力できる」選択式入力列（Excel 入力規則相当・ReadyCrew 要件）を提供する。あわせて列タイプメタの基盤（mount オプション `columnTypes`＋Internal な ColumnTypeRegistry）を導入し、DD-027-2/3 と P-07 判断材料の土台を作る。

## 背景・課題

- 親 DD-027 の決定事項（転記・2026-07-21 ユーザー確定）:
  - **①**: 列タイプは **grid 層のメタ**（mount オプション `columnTypes`＋Internal ColumnTypeRegistry）。core の CellScalar・protocol・snapshot は変更しない（値は string のまま）。
  - **②**: 候補は**列ごとの静的リスト**。検証は **commit 前（chokepoint `submitSetCells` 手前）の列 validator**。自由入力は既定不可（`allowFreeText` で許可可）。**paste/setData/リモート由来の非候補値は保持・表示**（拒否しない）。動的供給は拡張点メモのみ。
  - **⑤**: 公開は宣言的 mount オプションのみ。registry は Internal・consumer 向け登録 API 非公開（P-07 材料は親 Phase 4）。
  - **⑥**: `columnTypes` は両モード共通・mount 時固定（wrapColumns と同運用）。cell-commit/setData の値契約不変。共同編集での設定一致は利用側責務。
- 既存実装の接続点（2026-07-21 実調査）: 編集は 常駐 textarea（`integration-editor.ts`）＋編集状態機械（`packages/ime/editor-state-machine.ts`・5状態・I-1〜I-5 不変条件）の1種類のみ。keydown には**前段裁定 hook `interceptKeydown`**（DD-020-1 案X・Navigation 位相のみ・composition 中は必ず非消費）が既設。確定は `Commit` エフェクト→`ime-editing-session` →`config.submit`（=mount-controller の chokepoint `submitSetCells`。paste/範囲クリア/cut も同じ chokepoint・Undo 補償は別経路 `submitCompensation`）。beforeRevision 捕捉・行生存判定は `commit-bridge.ts`（`captureEditStartRevision`/`isRowLive`）が純関数で提供。playground は `?wrap=` の URL パラメータ先例あり（`apps/playground/src/integration/main.ts`）。
- DA 論点（親 Phase 0 より・本DDで扱う分）: (a) **grid 層メタ=サーバー検証なし** — 選択式制約はクライアント editor 経路のみで働く。悪意/設定不一致クライアントの非候補値は防げない（trusted internal 前提=憲章 §11。既知制約として記録）。 (b) **設定不一致クライアント混在** — 列タイプは描画/編集方式のみに影響し値は string のまま⇒文書状態・hash は乖離しない（表示/編集体験だけ異なる）。 (c) **validator と収束の両立** — validator を chokepoint 内に置くと paste/リモート経路まで拒否して②に反するため、**editor 経路（`createIntegrationEditor` へ渡す submit callback のラップ）だけ**に置く。

## 📐 実装前詳細化（親 Phase 0 判定=要。2026-07-21 実施済み・👀 ユーザーレビューは実装開始前の確認ゲートで）

**結論: `packages/ime`（editor-state-machine）・`integration-editor.ts` の composition 経路・常駐 textarea は無改変（T1 非該当の見込み）。** 接触点は mount-controller の既設 hook のみ。

- **モジュール構成（新規2・変更3）**:
  - 新規 `packages/grid/src/column-types.ts` — 公開型（`GridSelectColumnType` 等）の実体・Internal registry（`createColumnTypeRegistry(columnTypes, columnOrder)`）・validator（`validateEditorCommit(columnId, text)`）。純関数・DOM 非依存（TDD 対象）。P-07 材料となる v1 候補 I/F はここに集約。
  - 新規 `packages/grid/src/select-editor.ts` — ドロップダウン。**純粋状態コントローラ**（open/close/highlight/確定裁定・DOM 非依存=TDD 対象）＋**薄い DOM アダプタ**（listbox オーバーレイ・rAF placement 追従=badge と同方式）に分離。
  - 変更 `packages/grid/src/index.ts` — `GridCommonMountOptions.columnTypes?: Readonly<Record<string, GridColumnType>>` 追加（Experimental 0.x）。
  - 変更 `packages/grid/src/mount-controller.ts` — registry 生成・前段裁定拡張・dblclick 前段分岐・editor submit callback のラップ・ドロップダウン確定の submit。
  - 変更 `packages/grid/src/error-codes.ts` — 公開語彙 `value-not-allowed`（仮名・実装時確定）追加。
- **編集開始の横取り（選択式列・`allowFreeText:false` 時）**: dblclick=`editor.doubleClickCell` 呼び出し前に mount-controller で分岐／F2・Enter・文字キー・Alt+↓=既設 `interceptKeydown`（案X）の裁定拡張で消費してドロップダウンを開く。**既存契約（composition 中は必ず非消費・Navigation 位相のみ）を維持**するため状態機械は無改変。
- **フォーカス（I-5 維持）**: ドロップダウン表示中も focus は常駐 textarea のまま。↑↓/Enter/Esc は `interceptKeydown` で消費してコントローラへ転送。候補クリックは listbox の pointerdown（preventDefault で focus 維持）。
- **ドロップダウン確定のデータフロー**: open 時に `captureEditStartRevision` で beforeRevision 凍結 → 確定で `isRowLive` 確認 → SetCells 1件生成 → **既存 chokepoint `submitSetCells`**（Undo 記録・cell-commit 通知・自動行高が全て既存経路で成立）。行削除時は実行前拒否（`row-unavailable` 既存語彙・operationId 空文字規約・standalone は診断のみ）。
- **IME/自由 typing の残余経路（設計上の意図）**: IME composition による編集開始（compositionstart）は前段で横取りしない（横取り=IME 経路改変=T1）。よって選択式列でも textarea 編集自体は起こり得る。**非候補値の裁きは commit 時**: editor へ渡す submit callback を validator でラップし、非候補なら**未 submit・通知（拒否値を含む・`value-not-allowed`）・文書無変更**。ドラフト復元（編集継続）はしない=編集継続には状態機械改変（T1）が要るため不採用（親②「サイレント失敗なし」は通知で充足）。
- **T1 昇格条件（明記）**: 実装中に `editor-state-machine.ts`・`ime-editing-session.ts`・`commit-bridge.ts`・常駐 textarea の**変更**（利用は可）が必要と判明したら**停止して親DDへ記録し、親 Manual Gate T1（実IME 台帳5点＋変更固有手順）を必須化**する。
- **エッジケース**: 候補0件/重複候補（mount 時 config error=fail-fast）／ドロップダウン表示中の対象行削除（閉じて診断）／表示中のリモート値変更（開いたまま・確定は beforeRevision で OCC 裁定=既存 reject 経路）／画面外スクロール（placement null=閉じる。composition 不在ゆえ textarea の I-3 問題なし）／wrap 列との併用（配置は既存 placement 関数に委譲）。
- **テスト観点**: registry/validator 純関数・コントローラ状態遷移（TDD）／E2E=開閉・選択確定・両モード・非候補拒否・paste 保持／IME 回帰=不変条件スイート＋既存 IME E2E（無改変の証明）。

## 検討内容

- **拒否時 UX の選択**（📐 の帰結）: 「編集継続」案は T1（状態機械改変）を招くため見送り、「未 submit＋通知（値を含む）」を採用。将来編集継続が要件化したら別DD（親の既知制約へ記録）。
- **▼ インジケーター**: アクティブセルが選択式列のとき視覚手掛かり（▼）を DOM オーバーレイ（badge と同方式）で表示する（発見性のため in-scope・小）。全セルへの常時描画（Canvas 変更）は 27-3 の書式描画と重なるため対象外。
- **拡張点メモ（動的供給・P-07 材料）**: registry の候補供給を `readonly options: readonly string[]` で受ける形にし、callback/Promise 供給は I/F 拡張案として `column-types.ts` のコメント＋親 P-07 メモへ記載（実装しない）。

### 既知制約（DD-027-1）
- **grid 層メタ＝サーバー非検証**: 選択式制約はクライアント editor 経路のみで働く。悪意/設定不一致クライアントの非候補値は防げない（trusted internal 前提・憲章 §11）。paste/setData/リモート由来値も保持される（決定②）。
- **設定不一致クライアント混在**: 列タイプは編集/表示方式のみに影響し値は string のまま⇒文書 hash・収束は乖離しない（表示/編集体験だけ異なる）。
- **allowFreeText:true はドロップダウン非提供**（自由入力のみ）: リスト提示＋自由確定（Excel 準拠）は仕様拡張として見送り（Fable 5 見送り①）。→ 親 P-07「設定だけで実現できなかった要求」候補。
- **空セルで Enter→Enter → 先頭候補が書き込まれる**（AC どおり）: 発見性トレードオフ。親 Phase 4 Manual Gate M1 のスモーク観点（Fable 5 見送り②）。
- **数値/日付形の候補は使用不可**（mount 時 fail-fast）: 値は表示文字列で round-trip する必要がある（決定⑥）。ステータス等の文字列候補が対象。
