## 目的

Excel 風のテキスト表示を再現する:
1. **オーバーフロー**: A1 に長い文字列・B1 が空なら、A1 のテキストが B1（以降の連続空セル）へはみ出して表示される。B1 に値が入ればはみ出しは止まりクリップされる。
2. **折り返し**: 列に「折り返す」プロパティが設定されていたら、はみ出さずセル内で折り返す。**折り返しで収まらない行は必要な高さへ自動拡張**（Excel 同様）。

## 決定事項（仕様確定済み）

- **D1 wrap は列単位**（ユーザー確定）: `GridMountOptions.wrapColumns?: readonly string[]`（ColumnId）。セル単位書式モデルは Stage 2（stage2-backlog 記録済み）。実行時切替APIは今回見送り（必要になったら追加・mount option のみ）。
- **D2 オーバーフロー仕様（Excel 準拠・最小）**: 対象=**左寄せ描画の文字列セル**のみ（数値=右寄せは対象外と明記）。右方向の連続空セルへ表示を延長し、非空セルの手前・pane（固定列/ヘッダー）境界・viewport clip で止める。wrap 列のセルはオーバーフローしない（折り返す）。**データは不変・描画のみ**。
- **D3 可視走査の拡張**: 可視範囲の**左外**の非空セルからの流入を描画するため、行ごとに可視開始列から左へ最大 **20 列**遡って直近の非空文字列セルを探索（chunk-store 範囲クエリ）。20 列を超える流入は描画しない（**境界として明記**・Excel too は実質無制限だが性能予算優先。予算に余裕があれば Codex/実測後に拡大可）。
- **D4 折り返しの行分割**: 文字単位（CJK 前提・単語境界なし）。`measureText` キャッシュ（既存 TextMetricsCache）を使い、**値×幅キーの行分割キャッシュ**を新設（LRU・上限件数明記）。
- **D5 自動行高（Excel 風）**: wrap 列に非空値を持つ行の必要高 = max(各 wrap セルの行数)×lineHeight＋padding。再計算トリガー = ①bootstrap 直後の一括計算 ②セル値変更（ローカル楽観適用・リモート適用・rollback/replay 後） ③列幅変更（DD-012-4 の layout 変更） ④wrap 設定は mount 時固定。**手動リサイズ済みの行は手動値を優先**（Excel 同様・DD-012-4 の手動 override と区別するため「自動高」は別レイヤで保持し、手動 override があれば適用しない）。
- **D6 初期一括計算の性能ゲート**: 5万行シード（wrap 列非空 10万セル級）での bootstrap 後一括計算の所要を実測し、**予算（目安 ≦200ms・要実測確定）内なら採用**。超えるなら「可視行＋overscan のみ計算・スクロールで追補（totalSize が漸進更新される制約を境界として明記）」へフォールバック。判断は実測値を DD に記録して行う。
- **D7 自動行高と layout イベント**: 自動高は **`layout` GridEvent に含めない**（利用側が保存すべきは手動 override のみ。自動高は環境・フォントで再現される導出値）。
- **D8 features.json 更新**: 完了時に「Excel風テキスト表示（オーバーフロー・折り返し）」を提供中で追加＋デモシナリオ（長文入力→はみ出し→隣に値→クリップ→wrap列で折り返し・行高自動拡張）。

**設計要点:**
- オーバーフロー対象=左寄せ文字列のみ（数値=右寄せ・wrap 列は除外）。右方向の連続空セルへ延長し、非空セル手前は `fitText` で省略（AC2）、可視端まで空なら全文＋pane clip（AC1）。左外流入は行ごと最大20列遡り、pane 境界（frozenColCount）で停止（D3）。

**D6 bootstrap 一括計算 実測（node・synthetic measure＝13px latin 概算 7px/char・50,000行）:**
| wrap 非空セル | 値 | batch 所要 |
|---|---|---|
| 2,500（5%密度・perf seed 相当） | unique | 137ms |
| 10,000（20%密度） | unique | 141ms |
| 50,000（全行 wrap・最悪） | unique | 1,121ms |
| 50,000（全行 wrap） | 同一値（cache hit） | 205ms |
- **D6 判定=full batch 採用**（フォールバック不採用）。理由: ①**perf 回帰計測の scroll シナリオ（5万行）は wrap 列を使わない → 一括計算 0ms（no-op）→ scroll p95/redraw に影響なし**。②現実的密度（≤10〜20k wrap セル）は ~140ms で予算（≦200ms）内。③予算超過は「5万行すべてに一意の長文を wrap」する非現実的最悪ケース（1,121ms）に限られ、bootstrap 一度きり。可視域フォールバックは totalSize 漸進更新でスクロール飛びの副作用があり、現実シナリオを害するため見送り（Stage 2 で顕在化したら再検討）。境界として本 DD に明記。
- インクリメンタル（編集）経路は変更行のみ再計算＝O(1) 相当。
